Chapter 1. Get the Zapatec DHTML Tabs files

1. Download the Zip file

The DHTML Tabs files are bundled in a zip file. Click on the download link in the Tabs section of Zapatec web site, and follow the instructions to download the file.

Save the file (zptabs.zip) to your website root folder on your computer or server.

2. Extract zptabs.zip

  1. On your computer, go to the folder where you saved the file zptabs.zip.

  2. zptabs.zip contains a folder called zapatec that holds all of the files you will need to create your tabs.

  3. Double-click zptabs.zip to extract it.

Chapter 2. Quick Startup

If you have not done so, follow the instructions in section 1.1. to download and extract the Tabs files.

1. Set Up Your Tabs

In your web editor (Dreamweaver, UltraEdit, etc.), open the web page where you want to place the tabs. Position the cursor before the ending </head> tag. Paste the following style path and script paths before the ending </head> tag:

<!-- Javascript Zapatec utilities file -->  
<script type="text/javascript" src="zapatec/utils/zapatec.js"></script>
<!-- Javascript file to support tabs -->
<script type="text/javascript" src="zapatec/zptabs/src/zptabs.js"> </script>

<!-- optional Javascript file to support wizard -->
<script type="text/javascript" src="zapatec/zptabs/src/zptabs-wizard.js"></script>

<!-- optional Javascript file to support accordion tabs -->
<script type="text/javascript" src="zapatec/zptabs/src/zptabs-accordion.js"></script>

1.1. Path Warning

If you save the web page containing the tabs in the zptabs folder, these lines will work without any changes. If, however, you save your web page in another location (for example, directly under your website root folder), you need to adjust the path for these files.

1.2. Upgrade Warning

If you have been using a version of Zapatec tabs that is older than version 2.0 and are now migrating to the current version, you need to adjust the header files.

The older utils.js, transport.js, zpwidget.js are no longer supported. To upgrade, replace the old lines:

<script src="utils/utils.js" type="text/javascript"></script> 
<script src="utils/transport.js" type="text/javascript"></script> 
<script src="utils/zpwidget.js" type="text/javascript"></script>

with the following ones:

<script type="text/javascript" src="zapatec/utils/zapatec.js"></script>

The new zapatec.js file includes facilities to handle AJAX communications that used to be provided by the obsolete transport.js.

In addition, the use of the LINK element for CSS inclusion in pre-2.0 versions is no longer supported. For example, if you used the rounded.css in the HEAD of the HTML page your pre-2.0 version as in:

<link href="themes/rounded.css" rel="stylesheet" />

you should delete this line. Instead, set the CSS using the theme option in the Zapatec.Tabs constructor.

See the theme configuration option for details.

2. Attach the Zapatec Tabs Object to your Presentation

All you need in order to create Zapatec Tabs is two DIV elements - a tab bar and a content pane. The tabs implementation can get your content from a DIV in your document, as described in this section, or from files on a remote server accessed with Zapatec Transport.

Here is an example that lays out a web page with tabs. The tab bar is placed above the content. There are 2 tabs along with their content specified in HTML.You can put any kind of HTML after the label tag in tab body and it will be the content of your tab.

<div id="tabBar" style="width: 400px"></div>
<div id="tabs" style="height: 200px">
  <div id="photo">
    <label accesskey="h" title="Photo">P<u>h</u>oto</label>
    Photo Tab Contents
  </div>
  <div id="music">
    <label accesskey="m" title="Music"><u>M</u>usic</label>
    Music Tab Contents
  </div>
</div>

<script type="text/javascript">
  var objTabs = new Zapatec.Tabs({
    tabBar: 'tabBar',
    tabs: 'tabs'
  });
</script>

3. Copy your files to your web server

Using your editing or FTP program, copy or move your web page and the entire tree folder to the root of your website.

4. Test the tabs application

Using your web browser, navigate to the web page that you created to include the Zapatec DHTML Tabs. Congratulations! You have set up the most basic version of the Zapatec DHTML Tabs!

Chapter 3. Tabs

When you create tabs, you can pass a set of configuration options that affect its behavior.

A few configuration options are required:

  • theme

  • tabBar option that specifies tab bar element

  • tabs option that specified tab content pane element

Other configurations are optional.

1. Tabs Options

This section describes all supported configuration options you can pass when creating a new Zapatec Tab object.

1.1. tabBar (string or HTMLElement)

The tabBar option is a string or html DOM element reference, usually a DIV.

The tab bar is used to show the tabs that are currently opened. There is a DIV in the tab bar for each tab that is created. There is a text inside this DIV that is called tab label. When you click this label the tab becomes active.

It is up to you to set tab bar location relative to the tab content. You can place the tab bar above, alongside, or below the content.

To set tab bar above the content you should use the following layout:

<div id="tabBar"></div>
<div id="tabs">
...
</div>

To set tab bar below the content use: 

<div id="tabs">
...
</div>
<div id="tabBar"></div>

To set tab bar to the right of the content you can use:

<table>
<tr><td>
<div id="tabs">
...
</div>
</td><td>
<div id="tabBar"></div>
</td></tr>
</table>

The tabs theme that you use needs to support the type of positioning you choose. The themes currently provided by Zapatec are designed for tab bars positioned above the content. You can always create custom themes (CSS) to make tabs look good with your custom tab bar layout.

1.2. tabs (string or HTMLElement)

The tabs option is a string or html DOM element reference, usually a DIV. It is the element where the actual tabs content is dispayed.

The tabs content pane contains one DIV for each tab but only one of them is visible at a time. The currently visible one is called the active tab.

1.3. tabType (string, default: "div")

The tabType option specifies what kind of tabs you need. Setting this option to "div" will put tab content in a div. If you set it to "iframe", an iframe will be used for the tab content.

1.4. overflow (string, default: "hidden")

The overflow option specifies the overflow html attribute that is to be applied to tab content pane. Default is "hidden". Supported options are "visible", "auto" and "hidden". Set this option to "auto" to make scrollbars appear automatically when needed.

1.5. theme (string, default: "default")

The theme option specifies the CSS used to present tabs.

The string argument is the name of the theme:

theme : 'winxp'

or the absolute or relative path to the theme file:

theme : '../themes/winxp.css'

The theme name is case-insensitive. Some themes are provided in the themes folder.

If the theme option is not specified, the default.css file in the themes folder is used.

To use your own CSS throughout the enterprise without specifying the theme option for each page, you can rename your own CSS file "default.css" and move it to the themes folder.

1.6. themePath (string, default: path to "zptabs/themes" directory)

This is the path to the themes directory. If the path to the theme is passed as the argument to the theme option, the themePath is not needed.

If you use standard themes, the themePath is not needed.

1.7. ignoreUrl (boolean, default: false)

The tabs add a URL fragment "#tabId=activaTabId" to the page URL to create an unique URL for each tab. This way, if a user bookmarks the page, he will return to the tab that was bookmarked.

This is not always the desired behavior, since you may want the user to start at the first tab every time.

The ignoreUrl boolean indicates whether to ignore the #tabId section in the URL of a tab. To ignore the #tabId section of the URL, set ignoreUrl to true.

1.8. changeUrl (boolean, default: true)

The changeUrl boolean indicates whether browser location hash is to be changed on every tab change to reflect the current tab (as described above). If set to true, #tabId will be added to browser URL on every tab change. If false, tabs won't modify browser URL.

1.9. tab2tab (boolean, default: false)

The tab2tab option specifies if active tab should be changed by pressing the TAB key. Set this option to true to be able to navigate through tabs using the TAB key on your keyboard.

1.10. showEffect (string or array)

The showEffect defines animation effects to apply on tab content pane when a tab is made visible. Effect can be a string or an array. See Zapatec.Effects.init for more details. No effect will be applied if this option is not specified.

If you use this option you will have to include the following script file manually 

<!-- Javascript Zapatec Effects file -->
<script type="text/javascript" src="zpeffects/src/effects.js"></script>

1.11. showEffectSpeed (number)

The showEffectSpeed number indicates how fast should an effect be applied. See option above.

Here's an example of putting effects options to work.

var objTabs = new Zapatec.Tabs({
  ...
  // Make new tab fade-in when activated
  showEffect: "fade",
  // Set animation speed to 100ms
  showEffectSpeed: 100
});

1.12. mouseOverChangesTab (boolean, default: false)

The mouseOverChangesTab option specifies if tabs are to be changed by merely hovering over them with the mouse. When this option is set to true there is no need to click in order to make a tab active.

1.13. refreshOnTabChange (boolean, default: false)

The refreshOnTabChange boolean indicates if tabs are to reload from server each time they are to become active. When this option is set to true, tab contents is re-fetched every time a tab is activated.

1.14. shouldLoadOnInit (boolean, default: false)

The shouldLoadOnInit option indicates if tab contents is to be pre-fetched during tabs creation.

1.15. activeTabId (string)

The activeTabId option specifies the tab that is to become active after successful Tabs initialization. If not specified the first tab will be made active by default.

1.16. onInit (function)

The onInit option specifies a function called when the tabs are created. You can use it to perform application-specific initialization.

The function definition could look like this:

function callbackOnInit() {
}

1.17. onTabChange (function)

The onTabChange option allows you to set a callback that is called after the focus has left the old tab and is in the new one. The respective tab IDs are passed in one argument called objArgs.

The function definition could look like this

function callbackOnTabChange(objArgs) {
  // Get old tab ID string
  var oldTabId = objArgs.oldTabId;
  // Get new tab ID string
  var newTabId = objArgs.newTabId;
}

1.18. onBeforeTabChange(function)

The onBeforeTabChange option allows you to set a callback that is called when the user has indicated a desire to leave a tab. Return false in the callback to prevent tab change, return true to allow it.

A common use of onBeforeTabChange is to validate the contents of a tab and return false if the user needs to fix some errors before moving on to another tab.

The function definition could look like this

function callbackOnBeforeTabChange(objArgs) {
  // Get old tab ID string
  var oldTabId = objArgs.oldTabId;
  // Get new tab ID string
  var newTabId = objArgs.newTabId;
  // Allow tab change
  return true; 
}

2. Tabs Public Methods

Use public methods to do different manipulations with tabs using script.

To call public methods use the following syntax:

objTabs.methodname(argument);

2.1. changeTab(string)

Changes the currently active tab. Accepts one argument - id of the tab which will become active.

2.2. addTab(object)

Adds new tab. Accepts one argument - tab object which will be added to tabs.

For example:

  objTabs.addTab({
    id: 'myTabId',
    linkInnerHTML: '<u>M</u>y tab',
    title: 'My Tab',
    url: 'content-game.html'
  });

2.3. setVisible(boolean)

This method applies to single tab, not to whole tabs object. That is why it should be called like this:

objTabs.getTab('tabId').setVisible(false);

Its only argument is a boolean value which states if tab object is to be visible.

2.4. removeTab(string)

Removes the specified tab. Accepts one argument - id of the tab which will be removed from tabs.

3. Tab Options

You can set up labels and contents of the tabs either directly in HTML or using Javascript and AJAX. See section Attach the Zapatec Tabs Object to your Presentation. The following section explains how to create them using Javascript and AJAX.

Using this approach, you define the tabs in Javascript and fetch the data with AJAX, using an URL to point to the contents for each tab. The content is fetched from web server and inserted into the tab.

There are several advantages to this approach:

  • The main page does not hold any content. It holds only the definition of the tabs structure.

  • This is a more modular approach in which the contents of each pane are stored in a separate file.

  • The contents of the panes are fetched on an as-needed basis when the user requests them.

Here is an example of the tabs implemented in Javascript:

var objSource = {
  tabs: [
    {
      id: 'photo',
      linkInnerHTML: 'P<u>h</u>oto',
      accessKey: 'h',
      title: 'Photo',
      url: 'content-photo.html'
    },
    {
      id: 'music',
      linkInnerHTML: '<u>M</u>usic',
      accessKey: 'm',
      title: 'Music',
      url: 'content-music.html'
    }
  ]
};

var objTabs = new Zapatec.Tabs({
  tabBar: 'tabBar',
  tabs: 'tabs',
  source: objSource,
  sourceType: 'json'
});

Some options can be specified for an individual Tab as well as for Tabs. These are:

  • tabType

  • overflow

  • changeUrl

  • tab2tab

  • refreshOnTabChange

  • shouldLoadOnInit

Their meaning is the same as for whole Tabs object. They take precedence over the same options specified in Tabs. This way you can pass common option values to Tabs and override them when needed for an individual Tab.

Here are the options that are new to Tab:

3.1. id (string)

The id option specifies an unique string id for the tab. It is optional; if not defined, the tab generates an unique id automatically. You can use this id to get the tab instance.

var objTab = objTabs.getTab('photo');

3.2. index (number)

The index option specifies where tab is to be inserted relative to other tabs. It is optional; if not defined, a tab is appended after the last tab. This is a zero based index. Once the tab is created you can easily check what index it was inserted at.

var objTab = objTabs.getTab('photo');
var tabIndex = objTab.index;

3.3. linkInnerHTML (string)

The linkInnerHTML option specifies html to be used as a tab label. We use this option in the example above to set the tab label with our shortcut letter underlined.

3.4. accessKey (string)

The accessKey option specifies a keyboard shortcut for the tab. For example, the letter ’m’ is used for the Music tab above. The user can activate the tab by pressing ALT + access key. When defining a keyboard shortcut, be aware that the shortcut you choose might not work if it conflicts with a browser shortcut.

3.5. content (mixed)

The content option specifies tab content as an html string, URL or a DOM reference. See Zapatec Pane for more details as it is used internally by Tab to hold the actual content.

3.6. contentType (string)

The contentType option specifies type of content. Valid options are "html", "html/text", "html/url". See Zapatec Pane for more details.

3.7. url (string)

The url option specifies a relative or absolute URL where tab contents should be fetched from. This is equivalent as setting content option to the url string and contentType to "html/url". This is a legacy option that was left in for compatibility.

3.8. title (string)

The title option specifies a title html attribute to be applies to the label anchor. It acts as a tooltip of the tab label in some browsers. It is useful for specifying the title when it is broken up by HMTL formatting.

Chapter 4. Wizard

Wizard is a separate widget that inherits Tabs. It supports all options that Tabs does and some new ones.

You can easily build a wizard by combining two components, the Zapatec AJAX Tabs and the Zapatec AJAX Form.

Because it uses Form, before creating a wizard add the following to the headers:

<script type="text/javascript" src="zapatec/zpform/src/form.js"></script>

The typical applications that use a wizard are a shopping cart or a loan application in which the user can navigate back and forth through the different sections of the wizard.

Each tab would contain a form, which validates input data for the fields in the tab. Before allowing the user to continue on to the next tab, all required fields in the current tab would have to be validated. On the other hand, users could back up to the previous tab without requiring validation in the current tab. Such a wizard can easily be implemented using Zapatec Form and Zapatec Tabs.

One of the tabs can play the role of the Submit button. When a user clicks that tab, the following operations take place:

  1. All previous tabs are validated in order. If the validation fails in any tab, the user is taken to the tab that generated the error.

  2. If all forms on the previous tabs are valid, the data gathered from all wizard forms is submitted to the server.

  3. Depending on the server response to the submission, the onError or onSuccess callback is invoked.

When constructing a wizard, make sure that the fields throughout the wizard have unique names.

Since a wizard is the combination of the Zapatec AJAX Tabs and the Zapatec AJAX Form, we suggest that you should first become familiar with both of these components prior to creating a wizard.

1. Wizard Options

The wizard supports all the standard Tab options described above. Additionally, it supports the following options:

1.1. action (string)

This is the action attribute value of the hidden form. It specifies the URL to which you submit the form. The action option is required to submit data to the server. In some rare cases there is no need for the application to submit data or the data is submitted by an external script. In those cases the action option should be omitted.

1.2. submitTabId (string)

This is the id of tab that handles the submit result.

1.3. method (string, optional)

Use ’GET’ or ’POST’ to submit the form.

1.4. contentType (string, optional)

This is the content type for the POST method. Valid values are "div" or "iframe".

1.5. formThemePath (string, optional)

This is the relative or absolute URL to the form themes directory. See the Form documentation for details.

1.6. formTheme (string, optional)

Sets the theme for the Form. See the Form documentation for details.

1.7. formStatusImgPos (string, optional)

This option controls the position of the status image in relation to the field. See the Form documentation for details.

1.8. showErrors (string, optional)

This option controls the relative position of the error message in relation to the invalid field. See the Form documentation for details.

1.9. onError (function, optional)

This is the callback function to invoke on error. It receives the following object as an argument: 

{
  tabId: [string] tab id
  serverSide: [boolean] true if this is server response or false if validation result,
  generalError: [string] Human readable error description,
  fieldErrors: [
    {
      field: [object] field element object,
      errorMessage: [string] Human readable error description
    },
    ...
  ]
}

See Zapatec.Form submitErrorFunc configuration option for details.

1.10. onValid (function, optional)

This is the callback function to invoke after validation is passed. It receives the following object as an argument: 

{
  tabId: [string] tab id
}

See Zapatec.Form submitValidFunc configuration option for details.

1.11. onSuccess (function, optional)

Callback function to call after form was submitted and "success" response received from server. Receives object passed from server as argument.

See Zapatec.Form asyncSubmitFunc configuration option for details.

2. Adding Forms to the Wizard

To indicate that a Zapatec form should be included in the wizard, add the zpForm class to the form class attribute.

Each tab can contain only one Zapatec form. If there are multiple forms with zpForm in the class attribute, only the first form is used for validation. The others are ignored. You can, however, put additional non-Zapatec forms in the tab.

Form fields should be defined according to the Form rules.

A tab definition containing a form has the following structure: 

<div id="tab-begin">
  <label>The first tab</label>

  ... tab content ...

  <form class="zpForm">

    ... form fields definition ...

  </form>

  ... tab content ...

</div>

Here is a sample tab definition for a wizard: 

<div id="productTab">
  <label>Product Info</label>
  <div style="margin-bottom: 10px">
    <strong> In the Website field</strong><br />
    For a Single Server license, enter the full url of the server.<br />
    For a Domain Wide license, enter the domain name.<br />
    For an Unlimited license, enter the primary domain name.
  </div>
  <form id="productForm" name="productForm" class="zpForm" style="margin: 0px">
    <div id="productTabError" style="display: none; padding: 5px; font: bold 14px arial, sans-serif; color: #f00"></div>
    <table>
      <tr>
        <td><label class="zpFormLabel" id="pWebsiteLabel" for="pWebsite">Website</label></td>
        <td><input class="zpFormRequired" size="40" maxlength="1024" id="pWebsite" name="pWebsite" type="text" /></td>
      </tr>
      <tr>
        <td><label class="zpFormLabel" id="pNumberLicensesLabel" for="pNumberLicenses">Number of Licenses</label></td>
        <td><input class="zpFormRequired zpFormInt" value="1" size="10" maxlength="11" id="pNumberLicenses" name="pNumberLicenses" type="text" /></td>
      </tr>
      <tr>
        <td><label class="zpFormLabel" id="pLicenseTypeLabel" for="pLicenseType">License Type</label></td>

          <option value="Suite Single Server">Suite Single Server ($399) </option>
          <option value="Suite Domain Wide">Suite Domain Wide ($2400) </option>
          <option value="Suite Unlimited">Suite Unlimited ($3700) </option>

        </select></td>

  </form>
</div>

3. Initializing the Wizard

The following code initializes the wizard. The action and submitTabId configuration options are required.

var objWizard = new Zapatec.TabsWizard({

  // Tab bar div id
  tabBar: 'tabBar',

  // Tabs container div id
  tabs: 'tabs',

  // Tabs theme path
  theme: '../suite/zptabs/themes/red.css',

  // Before tab change event handler
  onBeforeTabChange: funcOnBeforeTabChange,

  // After tab change event handler
  onTabChange: funcOnTabChange,

  // Ignore "#tabId" part of URL to prevent opening other tab than the first right after initialization
  ignoreUrl: true,

  // Server script URL
  action: 'order.php',

  // Id of tab which submits data to the server
  submitTabId: 'orderTab',

  // Submit method
  method: 'POST',

  // Forms theme
  formTheme: '../suite/zpform/themes/winxp1.css',

  // Show error message after the field
  showErrors: 'afterField',

  // Error handler
  onError: funcError,

  // Valid handler
  onValid: funcValid,

  // Success handler
  onSuccess: funcSuccess

});

The onBeforeTabChange and onTabChange configuration options can be used to define callback functions that perform certain tasks before and after the user moves to a different tab. For example, they can control buttons behaviour, as in the following example: 

// Before tab change event handler
var funcOnBeforeTabChange = function(objArgs) {
  if (objArgs.newTabId == 'orderTab') {
    // Set "Next" button to submit PayPal form
    document.getElementById('buttonNext').onclick = function() {
      document.forms._xclick.submit();
    };
  } else {
    // Set "Next" button to change tab
    document.getElementById('buttonNext').onclick = function() {
      objWizard.nextTab();
    };
  }
  return true;
};

// After tab change event handler
var funcOnTabChange = function(objArgs) {
  // Show "Prev" button
  if (objArgs.newTabId != 'productTab') {
    document.getElementById('buttonPrev').style.display = 'inline';
  } else {
    document.getElementById('buttonPrev').style.display = 'none';
  }
};

The onError and onValid configuration options can be used to define callback functions that perform certain tasks when errors occur during form validation. For example, they can show or hide the error message: 

// Error handler
var funcError = function(objArgs) {
  var objErrorDiv = document.getElementById(objArgs.tabId + 'Error');
  if (!objErrorDiv) {
    return;
  }
  objErrorDiv.innerHTML = formErrorMessage(objArgs);
  objErrorDiv.style.display = 'block';
};

// Valid handler
var funcValid = function(objArgs) {
  var objErrorDiv = document.getElementById(objArgs.tabId + 'Error');
  if (!objErrorDiv) {
    return;
  }
  objErrorDiv.style.display = 'none';
};

This example shows how a formErrorMessage function parses the error object and constructs the error message: 

function formErrorMessage(objArgs) {
  var strMessage = objArgs.generalError + '<br />';
  if (objArgs.fieldErrors) {
    for (var iField = 0; iField < objArgs.fieldErrors.length; iField++) {
      var strFieldName = objArgs.fieldErrors[iField].field.name;
      var objLabel = document.getElementById(objArgs.fieldErrors[iField].field.name + 'Label');
      if (objLabel) {
        strFieldName = objLabel.innerHTML;
      }
      strMessage += strFieldName + ': ' +
       objArgs.fieldErrors[iField].errorMessage + '<br />';
    }
  }
  return strMessage;
}

The onSuccess configuration option can be used to define a callback function that performs certain tasks when wizard data is submitted and a success response is received from the server.

var funcSuccess = function(objArgs) {
  ... do something with objArgs object received from the server ...
};

Chapter 5. Accordion

Accordion is a separate widget that inherits Tabs. It supports all options that Tabs does.

Accordion is Tabs with a special slidding effect during tab change. Accordion ignores the tabBar option so it is no longer required as in Tabs.

All you have to do in order to use Accordion Tabs instead of regular Tabs is to change:

var objTabs = new Zapatec.Tabs({
  ...
});

into this

var objTabs = new Zapatec.AccordionTabs({
  ...
});

Chapter 6. Scenarios

1. Next / Previous tab

You can also add optional navigation buttons to the page to move the focus to the next and previous tabs. You can put these buttons anywhere on the page. 

<button onclick="objTabs.prevTab()" accesskey="p">&laquo; <u>P</u>rev.</button>
<button onclick="objTabs.nextTab()" accesskey="n"><u>N</u>ext &raquo;</button>
